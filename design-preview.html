Shuckler Design System — v2 Full Redesign

Shuckler Reborn

Analog warmth. Digital precision. A personal music app that feels like it was made by hand — because it was.

Foundation

Seven design principles

Every decision traces back to one of these.

01
Music moves the UI
Audio amplitude drives subtle animations everywhere — not decorations bolted on top.
02
Rectangles, not circles
Music lives in rectangles: album covers, cassette inserts, jewel cases. Circles crop the artist's composition.
03
Warmth over neutral
#0C0A07, not #000000. A whisper of warmth makes the app feel like listening under a lamp.
04
Gestures are first-class
Expand, dismiss, skip — via continuous gestures. The UI feels physically held.
05
The DM type system
DM Serif Display (emotional) + DM Mono (data). Same type family — intentional, cohesive, no cursive.
06
Art as the only accent
Album Palette tints every surface. An advantage no commercial streaming service can copy.
07
Your listening, visible
History, stats, and scrobbling make the app feel like a living record of your taste — not just a player.

New Features

What gets added to the app

These aren't just design changes — they're features that make Shuckler meaningfully better.

Backend
Last.fm Scrobbling
Authenticate with Last.fm in Settings. Every song played past 50% is scrobbled automatically. Your listening history lives on your Last.fm profile forever.
Backend
Spotify Metadata Enrichment
Uses the existing SpotifyAuthManager to silently pull artist photos, biographies, and similar artists for songs already in your library. No extra effort after initial auth.
Backend
Listening History + Stats
Every play event is stored locally in Room (song, timestamp, ms played). Surfaces as "Recents" in Library and full stats on the Profile tab — weekly chart, top artists, all-time hours.
Backend
Smart Auto-Playlists
Generated from listening history via SQL queries: "Most Played," "Recently Added," "Long Sessions." No ML — just data you already have, surfaced intelligently.
Frontend
Artist + Album Pages
Tap any artist name or album art to drill into a dedicated page with a blurred palette hero, all songs, and complete tracklist with track numbers.
Frontend
Draggable Queue
Full queue screen accessible from Now Playing. Rows are draggable with spring-physics reordering. Now Playing card sits pinned at top with live amplitude bars.
Frontend
Line-Synced Lyrics
Accessible from the LYRICS chip on Now Playing. Active line renders in DM Serif Display at a larger size. Adjacent lines de-emphasize with opacity and size transitions.
Frontend
Crossfade + Sleep Timer
Configurable crossfade duration (0–10s) between tracks. Sleep timer with a simple countdown that fades volume before stopping — both in Settings.
Interaction Design

Gesture-first navigation

Core actions via continuous gestures. The UI responds as if physically held.

↑
Swipe up pill
Expands to Now Playing with a shared-element transition on the album art
↓
Swipe down
Collapses Now Playing — art springs back to pill thumbnail size
←→
Swipe album card
Left = next, right = previous. Card tilts with physics then snaps away
⟳
Long press track
Context menu spring-in from press point — queue, download, playlist, share
↔
Scrub progress
Drag on bar — thumb springs to finger, haptic at start and end
↕
Drag queue rows
Long press + drag to reorder queue with spring physics and haptic on lift
Implementation

What ships, and what's bold

Green = ship in one pass. Amber = requires more architectural care.

Safe — ship first
Design token system in Color.kt — all hardcoded colors replaced
Warm base #0C0A07 + full surface elevation hierarchy
DM Serif Display + DM Mono via Compose downloadable fonts
Album art: CircleShape → RoundedCornerShape(16.dp) everywhere
Shimmer skeletons replacing all progress spinners
Empty states with DM Serif copy and amber CTA
Download waveform card — remove full-width progress bar
Spring physics on all button presses (existing pattern)
DM Serif headers on Library, Search, Downloads, Queue
Artist page + Album detail screens (new nav destinations)
Listening history Room table + Stats screen
Last.fm scrobbling (OAuth + API calls)
Crossfade + sleep timer in Settings
Bold — requires care
Floating pill mini player — replaces bottom bar, layout architecture change
Shared-element transition pill → Now Playing (Compose SharedTransitionLayout)
Breathing glow ring driven by live FFT amplitude
Palette API tint propagating to all surfaces, not just player
Swipe-to-skip gesture on album card with velocity physics
Draggable queue reordering with spring-physics rows
Line-synced lyrics (requires timestamp data from lyrics provider)
Crossfade audio implementation in MusicPlayerService
Haptic feedback patterns matched to UI spring events
Spotify metadata enrichment — background sync, attribution UI
