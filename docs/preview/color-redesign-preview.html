Shuckler / Design Revision v2

Color & Animation Redesign

The problem: album art extracts a beautiful accent color, then leaves it in the Now Playing room. This redesign bleeds that color through the whole app, adds a layered ambient light model, and defines a concrete animation system. The palette tokens are unchanged — only where and how they're applied.

01 — The Core Problem
Ambient Album Atmosphere
Right now, LocalAccentColor is extracted from album art and tints a handful of specific elements: the progress bar, amplitude bars, the playing row wash, and Now Playing controls. That's it. The color dies at the Now Playing screen boundary.

The redesign extends the Palette color as ambient light — a subtle full-app color cast, as if the album cover is glowing behind the entire interface. Think: vinyl store mood lighting, not a spotlight. Every screen reflects the currently playing album, not just the player.

02 — Library Redesign
A Record Collection, Not a File Manager
The Library currently shows a flat list of tracks. The new direction organizes by album first — albums are the natural unit of a music collection. Albums get a shelf hero. Tracks are grouped under album headers. The section headers use the album's extracted color as a subtle tint, making each album feel like its own micro-world.

03 — Animation System
Six Named Animations
Each animation has a name, a trigger condition, and Compose code. These aren't decorations — each one is tied to a specific UX moment. The rule: if removing the animation doesn't confuse the user, the animation should not exist.
Ambient Transition
TRIGGER: Track changes
When the track changes, the ambient bloom color cross-fades from the old album's accent to the new one over 600ms. The transition uses animateColorAsState so it's interruptible. Never jarring — the room slowly changes color.
val ambientColor by animateColorAsState( targetValue = LocalAccentColor.current, animationSpec = tween( durationMillis = 600, easing = FastOutSlowInEasing ) ) // Apply as radial gradient overlay at 15% opacity // using a Modifier.drawBehind with canvas radialGradient
List Stagger Entrance
TRIGGER: Screen enters / filter changes
First 5 visible items enter with a staggered fade+slide (8dp Y, 30ms stagger, 280ms each). Items beyond index 4 appear instantly to keep scroll performance clean. Key stability is required — see prior pitfall learning.
@Composable fun TrackRow(track: Track, index: Int) { val visible = remember { MutableTransitionState(false) } LaunchedEffect(Unit) { delay(minOf(index, 4) * 30L) visible.targetState = true } AnimatedVisibility( visibleState = visible, enter = fadeIn() + slideInVertically { 24 } ) { /* row content */ } }
Tab Slide
TRIGGER: Bottom nav tab tap
⌂ Home
♫ Library
Tab switches slide content left or right based on tab index direction. No fade — the directional slide creates spatial memory. Users learn where Library is relative to Home because they always arrive from the same direction.
AnimatedContent( targetState = selectedTab, transitionSpec = { val dir = if (targetState.index > initialState.index) 1 else -1 slideInHorizontally { it * dir } + fadeIn() togetherWith slideOutHorizontally { -it * dir } + fadeOut() } ) { tab -> TabContent(tab) }
Press Feedback
TRIGGER: Any tappable element
PLAY ALL
Scale 0.94f on press using spring physics with medium bouncy damping. Applies to: track rows, chips, shelf items, album cards, nav items, and all buttons. This is the micro-feedback that makes the app feel alive — every surface responds to touch.
val scale by animateFloatAsState( targetValue = if (pressed) 0.94f else 1f, animationSpec = spring( dampingRatio = Spring.DampingRatioMediumBouncy, stiffness = Spring.StiffnessMedium ) ) Modifier.graphicsLayer { scaleX = scale; scaleY = scale }
Album Art Bloom
TRIGGER: Currently-playing album in shelf
The currently-playing album in the Library shelf gets a pulsing color bloom behind it — a soft radial gradient of the album accent that breathes at 2s intervals. Scale 0.95→1.05, opacity 0.4→0.9. No bloom on non-playing albums.
val bloomScale by animateFloatAsState( targetValue = if (isPlaying) 1.05f else 0.95f, animationSpec = infiniteRepeatable( animation = tween(2000, easing = FastOutSlowInEasing), repeatMode = RepeatMode.Reverse ) ) // Drawn as a blurred Box behind the artwork Box
Download Spring Collapse
TRIGGER: Download completes → library entry
DONE — Oblivion
When a download finishes, the waveform card flashes green, shows "DONE" for 1.5s, then spring-collapses vertically (scaleY 1→0, overshoot allowed) and disappears as the track inserts itself into the library list. The track arrival gets the stagger entrance.
AnimatedVisibility( visible = !downloadComplete, exit = shrinkVertically( animationSpec = spring( dampingRatio = Spring.DampingRatioMediumBouncy, stiffness = Spring.StiffnessLow ) ) + fadeOut(tween(200)) ) { WaveformDownloadCard(...) }
04 — Color Token Additions
Extending Color.kt
The existing token palette is correct. No colors are removed. Three new tokens added to support the ambient system, and the dynamic LocalAccentColor gets documented as the primary driver of ambient color rather than just the progress bar.
AmbientTint
derived from LocalAccentColor at 15% opacity
Full-screen ambient bloom — applied as a Modifier.drawBehind radial gradient over the Base background.
NEW
AccentWash
LocalAccentColor at 7% opacity
Playing row background wash. More restrained than the ambient bloom — this is for surface-level tinting of individual rows and section headers.
NEW
AccentChip
bg: AccentColor@12% · border: AccentColor@40%
Selected chip/pill background. Replaces the old static amber chip with a dynamically tinted one that matches the current album accent.
NEW
AlbumGroupAccent
AccentColor@5% bg + AccentColor@30% border-bottom
Album section header background when that album is currently playing. Gives each playing album its own micro-environment in the Library grouped view.
NEW
Kotlin additions to Color.kt
// ── Ambient / Dynamic Tints (derived from LocalAccentColor at runtime) ─────── // These are not static Color values — they are applied via Modifier.drawBehind // or as inline alpha-composited copies of LocalAccentColor.current. // // Usage: // val accent = LocalAccentColor.current // val ambientTint = accent.copy(alpha = 0.15f) // full-screen bloom // val accentWash = accent.copy(alpha = 0.07f) // row/section background // val accentChipBg = accent.copy(alpha = 0.12f) // selected chip bg // val accentChipBorder = accent.copy(alpha = 0.40f) // selected chip border // val albumGroupBg = accent.copy(alpha = 0.05f) // playing album header bg // ── Semantic tint helpers (add to a Theme.kt or AccentExtensions.kt) ───────── @Composable fun accentAmbient() = LocalAccentColor.current.copy(alpha = 0.15f) @Composable fun accentWash() = LocalAccentColor.current.copy(alpha = 0.07f) @Composable fun accentChip() = LocalAccentColor.current.copy(alpha = 0.12f)
05 — Implementation Plan
What Codex Implements, in Order
Four slices, each shippable on its own. The first slice gives the biggest visual uplift in 30 minutes of work. Later slices add polish.
Phase What ships Files touched Effort
1 — Ambient Full-screen accent bloom on all screens. AccentWash + AccentChip helpers. Mini player tint. NavGraph.kt, theme/Color.kt + new AccentExtensions.kt, MiniPlayerBar.kt, each screen scaffold ~2h
2 — Library Album View By Album filter chip, album-grouped list, playing album header tint, shelf bloom ring. LibraryScreen.kt, LibraryViewModel.kt ~3h
3 — Animation System PressScale modifier, stagger entrance for lists, tab slide transition, ambient color transition. NavGraph.kt (tab transitions), PressScale.kt (new util), stagger in TrackRow.kt, LibraryScreen.kt, HomeScreen.kt ~3h
4 — Polish Album bloom in shelf, download spring collapse, Lyrics line entrance animations. LibraryScreen.kt, WaveformDownloadCard.kt, PlayerScreen.kt ~2h
Phase 1 is the highest-leverage change: it touches every screen but requires no structural refactor — only a new composable wrapper in the NavGraph scaffold and a handful of helper functions. The result is an app that immediately feels warmer and more alive, even before the album view or animations land.
